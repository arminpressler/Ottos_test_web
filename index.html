How to change this website (hi Kimo!)

  1. Open the project on GitHub Visit the repo arminpressler/Ottos_test_web and make sure you are signed in.
  2. Edit the page Browse to index.html, click the pencil (Edit this file) button, and make your changes directly in the editor.
  3. Create a change proposal Below the editor, describe what you changed and select “Create a new branch and start a pull request.” Use a name like feature/your-change so it’s easy to track.
  4. Review the preview GitHub will open a pull request into the preview branch. Click the “Preview” link in the PR description (or visit the staging site) to confirm everything looks right.
  5. Publish Once the preview looks good, merge the pull request into preview. When you’re ready to ship to the live site, open a PR from preview to main and merge it after a quick smoke test.

Bonus: prefer working locally? Clone the repo, create a feature branch off preview, edit with your favorite editor, then push and open a PR. The README walks through the workflow in more detail.
